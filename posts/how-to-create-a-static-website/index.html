Create static website with Hugo

Hello everyone, My name is Mahmudul Hasan. This post is the first post on the blog. While after creating this website, I was thinking what can be written as post no. 1, and discovered that: its hard to come up with one good idea. However, I came up with this idea eventually — I’d like to share how I created this website. Here is why I think its better:

  • Maybe you know how to create one, or don’t but its always good to know other’s experience, right?

  • This content can actuall help someone trying to start her/his own blog or website with Github Pages.

  • I don’t have to display some !important contents :no_good:

So maybe this post’s title should be — Steps I used to create this site with Hugo and Github Pages. So, lets start.

Install and prepare static site with Hugo

Installation of Hugo is pretty strightforward, it can be found on their installation doc. As, I am using Macports on my Macbook Pro, I just put following in the command-line. By the way, this process isn’t listed on above installation page. :+1:

sudo port install hugo

Hugo should be installed globally as a command. After installtion process is complete, confirm that Hugo is installed by

hugo version
# output for me was:
# Hugo Static Site Generator v0.55.6 darwin/amd64 BuildDate: unknown

Now go to any directory you want your blog or website to reside, and put following.

hugo new site your-blog-name
cd your-blog-name

You should now see lot of automatically generated files. Mostly you should be focusing on

  • content — you write contents here
  • themes — themes goes here
  • static — files, images, etc goes here
  • public — publishable files goes here
  • and config.toml — site configuration here

Its important to install a theme at this point. For this website, I used my clone of One theme. I have made some changes on this theme which is available on my forks master branch now. I never did sent a PR.

git clone https://github.com/hasanlock/hugo-theme-one.git themes/one

However, a fair list of hugo theme collection is also available. Put all the time you need for your theme selection.

After this cloning process, add this theme to your blog’s configuration to activate it. I also did updated some other configurations. After those changes my config.toml looks like:

baseURL = "https://hasanlock.github.io/"
languageCode = "en-us"
title = "Mahmudul Hasan"
theme = "one"
enableEmoji = true
pygmentsCodefences = true # syntax highlighting
pygmentsStyle = "native"
[params]
    navigation = ["about.md"]

Configurations other than setting base url, site title, locale; are mostly for syntax highlighting. Here [params] section is used for site navigation links.

Start writing content

After settings up website or blog the thing to do is writing content, adding external link and images. I am going to give you a brief idea — how I do it.

Create new post

To generate a new post you enter following command:

hugo new posts/your-content-title.md
Write content with markdown

Hugo uses Markdown for content development and for writing Markdown I use Typora, which is a desktop software available for Win, Mac, and Linux.

Share link and image

Like everybody else, I use Imgur for image hosting. Just upload the image and hotlink it!

If you browse the folder structure you’ll find a static directory inside. This is the directory where you should put all your attachment or images. You are free to organize however way I suits you.

Hosting with Github Pages

So there is a dedicated github’s site for the explanation on how it works, so I am not going to dive in details. In short you can host static sites on github pages. There are similar services like Bitbucket Pages and Gitlab Pages. The following procedure is same for these services.

Launching the site locally

After you modified the contents, you would like to see the contents.

hugo server -D

User above command, it will block your terminal. Now open the browser of your choice, and open http://localhost:1313. Voilà! Now you should see your shiny new site.

This command uses live reload feature, and include draft contents. That means, you can check the content as you go.

Publish content

When you think that you are ready to publish your site, then first you need to generate publishable content by running:

hugo

then you’ll see changes in public directory inside the project directory. This is the publishable website you have.

Create github repository and publish

create a github PUBLIC repositroy named https://github.com/your-github-username/your-github-username.github.io, in this case your static website URL will be your-github-username.github.io. More details can be found here. After that do the following:

git add --all
git commit -m "Initial commit"
git push -u origin master

…and you’re done! point your browser to your-github-username.github.io and see your website or blog.

Thank you for reading. Good day.
